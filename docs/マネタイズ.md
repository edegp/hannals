# マネタイズ設計（車両課金モデル）

更新日：2025-12-13（JST）

対象：積み込み最適化（配送順 × 積載レイアウト）プロジェクト

---

## 0. 結論（重要事項）

- **課金方式は車両課金**で統一する
- 相手方へは「**1台あたり月7〜8万円の便益**」を前提に、
「月額サブスクでもこれだけ純メリットが残る」を**1台単位で**提示する
- **価格帯は 9,800〜19,800円 / 台 / 月**（便益の約11〜23%）を採用
- PoCは別価格で入口を作り、本契約へ移行させる
- **MVP段階では Starter プラン（9,800円 / 台 / 月）のみを開発・提供する**

---

## 1. 前提（プロジェクトのスコープ）

### 1.1 対象領域

- **ラストワンマイル（直接配達）**：EC宅配・企業向け小口
- 車両想定：**2tトラック**中心

### 1.2 KPI（プロジェクト宣言）

- **停車時間**（1ストップあたり）
- **出発遅延**（積み込み完了〜出発）

> 積載率はKPIにしない（ラストワンマイルでは指標としてズレやすい）
> 

---

## 2. 価値（便益）設計：1台あたりの金額

### 2.1 便益の基本ストーリー

- 本サービスは「**時間を生む**」プロダクト
→ 生まれた時間を
    - **コスト削減**（残業/稼働削減）
    - **増収**（追加配送）
    に転換できる

### 2.2 便益（ベースライン）

- **1台あたり月7〜8万円**の便益（試算ベース）を営業・提案の統一前提とする
- 便益の内訳（説明用）
    - (A) 停車時間・出発遅延の短縮による稼働削減（守り）
    - (B) 短縮時間分の追加配送による増収（攻め）
    - (C) 積み込みを低単価スタッフへ移管できる運用メリット（補助）

> 注：時給差（正社員 1,400〜1,500円 / バイト 1,226円）は小さいため、(C)は金額より運用価値（属人化排除・教育不要化）として前面に出す。
>

## 3. 誰が払うか（ターゲット顧客）

### 3.1 最優先ターゲット

- **倉庫会社（3PL）が自社車両を持ち、配送まで行う事業者**
    - 倉庫（積み込み）と配送（停車時間）の改善を**同じ意思決定**で動かせる
    - PoC成功率が高い（責任境界で止まりにくい）

### 3.2 代替ターゲット（次点）

- 地域配送会社（自社車両）
- 荷主（EC/メーカー）の物流部門（CLO文脈）

---

## 4. 収益モデル（車両課金 + 初期導入費 + オプション）

### 4.1 基本：車両課金サブスク

- **◯円 / 台 / 月** を基本単位とする
- メリット：相手に「1台あたりでROI」を提示しやすく、導入台数増で拡張しやすい

### 4.2 初期導入費（別建て）

- **200,000〜600,000円 / 拠点**
- 含む：
    - 車両設定（2tテンプレ/荷室・出入口）
    - 入力フォーマット整備（CSV）
    - 積載ルール定義（壊れ物、天地無用、上積み禁止など）
    - 現場教育（倉庫/配車/ドライバー）
    - KPI計測設計（出発時刻・停車時間ログ取り、比較設計）

> 現場プロダクトは導入支援が必ず発生するため、初期費用を無料にしない（採算・成功率の観点）。
> 

### 4.3 オプション（アップセル）

- **分析ダッシュボード強化**（便・車両・担当別の比較、改善レポート）
- **連携**（WMS/TMS/配送アプリ、バーコード/スキャン）
- **高度制約**（重量・重心・安全ルール強制、同梱禁止など）
- **監査/証跡レポート**（荷主報告、社内改善活動の証跡）

## 5. 価格設定（車両課金：確定案）

### 5.1 価格レンジの根拠

- 便益：**70,000〜80,000円 / 台 / 月**
- 価格：便益の**10〜20%程度**を目安に設定
→ 相手に「費用対効果」を即理解させ、かつ十分な粗利を確保

### 5.2 プラン（3段階）

> **MVP段階での提供プラン**  
> MVP段階では **Starter プランのみ**を開発・提供する。  
> Standard、Proは市場検証後に段階的に開発する。

要件定義の内容と一致

### Starter（PoC〜小規模導入）**【MVP対象】**

- **9,800円 / 台 / 月**
- 含む：CSV取込、車両テンプレ（2t）、積み付け提案、3D表示、基本ルール

### Standard（本命：推奨）**【将来機能】**

- **14,800円 / 台 / 月**
- Starter + **KPI簡易ダッシュボード**（停車時間/出発遅延）、便別比較、週次レポート

### Pro（連携・高度制約）**【将来機能】**

- **19,800円 / 台 / 月**
- Standard + API/スキャン連携、高度制約、権限管理、監査/証跡

    - 小規模でも運用サポートが発生するため最低ラインを設定

### 7.1 推奨PoC規模

- 期間：**4〜8週間**

    - **PoC費用の一部を初月に充当**（移行率向上）

## 8. ROI提示（営業トークの型）

### 8.1 1台あたりの純メリット（例）

- 便益：70,000〜80,000円 / 台 / 月

### Standardの場合（14,800円 / 台 / 月）

- 純メリット：**55,200〜65,200円 / 台 / 月**

### Starterの場合（9,800円 / 台 / 月）

- 純メリット：**60,200〜70,200円 / 台 / 月**

### Proの場合（19,800円 / 台 / 月）

- 純メリット：**50,200〜60,200円 / 台 / 月**

> 「月額◯円でも、御社は1台あたり月◯万円メリットが残る」という言い方を標準化する。
> 

### 8.2 メリットの出し方（説明の順序）

1. KPI改善（停車時間・出発遅延）
2. 時間短縮の総量（分/日/台）
3. ①コスト削減 + ②追加配送（増収）
4. 1台あたり月次便益（7〜8万円）
5. サブスク費用との差分（純メリット）

---

## 9. リスクと対策（価格設計に効くポイント）

- **便益が出ないと解約**
→ StandardでKPI可視化を標準化し、「改善活動」を継続できる設計にする
- **計測のブレ**（繁忙/曜日差）
→ PoCではA/Bを優先し、導入費に計測設計を含める
- **入力負荷が高いと運用が崩れる**
→ MVPはCSV中心＋欠損許容、現場負荷を最小化する
- **倉庫/配送が別会社だと詰まる**
→ 3PL＋自社車両を優先ターゲットにする
